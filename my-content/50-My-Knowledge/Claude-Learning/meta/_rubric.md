# Rubric — 筛选标准

筛选一个 Claude 相关 repo/文章时，按以下维度打分（每项 1-5）：

## 维度

| 维度 | 说明 | 权重 |
|------|------|------|
| **可复用性** | 能直接拿来用，不是 demo 级别 | 高 |
| **有示例** | 有具体例子，不是纯理论 | 高 |
| **新颖性** | 和已收藏的有差异，不是 awesome-list 二次包装 | 中 |
| **质量** | 逻辑清晰，格式规范，有 frontmatter 或结构 | 中 |
| **时效性** | 最近一年内有维护 | 低 |

## 快速判断

**直接拒绝（❌）**：
- 纯 awesome-list（只是一堆链接）
- 没有具体示例的理论文
- 只是 OpenAI prompt 改个模型名
- Stars > 1000 但内容空洞（虚假流行）

**优先收（✅）**：
- 带 frontmatter 的 .md skill，有 `<command-name>` / `<system>` 等结构
- Anthropic 官方 cookbook / 示例
- 展示具体 workflow / agent 编排模式的
- 有实测效果记录的

## 分数标准

- **4-5**：立刻 ingest，蒸馏进 notes
- **3**：ingest 原文，蒸馏笔记可简短
- **1-2**：只在 pending 里留一行链接，不占空间
