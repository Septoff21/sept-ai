# Morning Standup Brief Generator — Руководство пользователя

Как сформировать дейли-бриф из локальных файлов за две минуты.

---

## Быстрый старт

Самый быстрый способ получить бриф для стендапа:

1. Убедитесь, что в папке есть хотя бы один `.md` или `.txt` файл с заметками или задачами
2. Напишите: `"Утренний бриф для стендапа"` и укажите путь к папке
3. Получите структурированный бриф: Вчера / Сегодня / Блокеры / Вопросы

**Результат:** Готовый бриф в чате и файл `standup-brief-YYYY-MM-DD.md` в вашей папке.

**Время:** ~2 минуты

---

## Сценарии использования

### Сценарий 1: Разработчик готовится к командному стендапу

**Ситуация:**
Вы разработчик и ведёте дневные задачи в файле `work.md`. До утреннего стендапа 5 минут — нужно быстро вспомнить, что сделал вчера, что планируешь сегодня, и не забыть про блокеры. Листать файл вручную некогда.

**Что делать:**

1. Откройте Cowork-сессию с папкой, где лежит `work.md` (или запомните путь к папке)

2. Запустите скил: `"Утренний бриф для стендапа"`
   - Если рабочая папка не выбрана, укажите путь: например, `/Users/yourname/work/`

3. Дождитесь, пока скил просканирует файл и извлечёт элементы
   - `- [x]` попадут в «Вчера», `- [ ]` / `TODO` — в «Сегодня», `BLOCKER` — в «Блокеры»

4. Просмотрите бриф в чате
   - Проверьте, что блокеры зафиксированы корректно
   - Если что-то пропущено — добавьте в сохранённый файл вручную

**Ожидаемый результат:**

Вы получаете бриф примерно такого вида:
```
## Вчера (Выполнено)
- Проревьюил PR #22, оставил комментарии
- Починил нестабильный тест в CI

## Сегодня (План)
- Реализовать таймаут сессии (дедлайн завтра)
- Ответить на комментарии @Ивана к PR

## Блокеры
- Жду одобрения схемы БД от @Лида

## Вопросы / Жду ответа
- Уточнить у @DevOps доступ к стейджингу
```

Зачитываете бриф на стендапе — готово за 60 секунд.

**Почему это работает:** Вместо того чтобы пролистывать `work.md` каждое утро, вы получаете готовый бриф без ручной сортировки и риска забыть про блокер.

### Сценарий 2: Продакт-менеджер агрегирует заметки из нескольких файлов

**Ситуация:**
Вы продакт-менеджер, заметки разбросаны по нескольким файлам: `sprint-notes.md`, `meeting-log.md`, `backlog-ideas.md`. Перед стендапом нужно понять, что продвинулось, что запланировано на сегодня, и что мешает команде.

**Что делать:**

1. Сложите нужные файлы в одну папку (или используйте существующую папку проекта)

2. Запустите скил: `"Собери мои задачи и заметки для стендапа"`
   - Укажите путь к папке

3. Скил автоматически:
   - Отфильтрует файлы, изменённые за последние 48 часов
   - Извлечёт задачи, выполненные пункты и блокеры из всех файлов
   - Дедуплицирует элементы, встречающиеся в нескольких файлах

4. Просмотрите бриф
   - В футере «Источники» проверьте, какие файлы были использованы
   - Если важный файл не попал (нет недавних изменений), укажите явно: «Добавь также planning.md»

**Ожидаемый результат:**

Вы получаете сводный бриф из всех файлов:
```
## Вчера (Выполнено)
- Закрыли 3 задачи с дизайн-командой
- Обновили продуктовый роудмап Q2

## Сегодня (План)
- Подготовить демо для стейкхолдеров (дедлайн сегодня)
- Обсудить velocity спринта с лидом разработки

## Блокеры
- Ждём юридического согласования обновлённых Условий использования

## Вопросы / Жду ответа
- Нужен ответ от @Маркетинг по срокам кампании

---
Источники: sprint-notes.md, meeting-log.md
```

**Почему это работает:** Несколько файлов — один бриф. Не нужно открывать каждый файл и копировать вручную.

### Сценарий 3: Ведение лога стендапов по дням

**Ситуация:**
Вы хотите хранить историю стендапов — по файлу на каждый день — чтобы в конце недели подвести итоги или подготовить недельный отчёт. Вы уже несколько дней пользуетесь скилом и хотите убедиться, что брифы сохраняются и доступны.

**Что делать:**

1. Запускайте скил каждое утро: `"Дейли бриф"` — в одну и ту же папку

2. Скил автоматически сохраняет `standup-brief-YYYY-MM-DD.md` с датой в имени файла
   - Каждый день — отдельный файл: `standup-brief-2026-04-20.md`, `standup-brief-2026-04-21.md` и т. д.
   - Если бриф за сегодня уже существует — скил уточнит, перезаписать ли его

3. В конце недели откройте папку — там будет 5 файлов с брифами

4. Используйте недельные брифы для пятничного статус-апдейта или ретроспективы

**Ожидаемый результат:**

В папке накапливаются дейли-брифы:
```
/work-notes/
├── standup-brief-2026-04-18.md
├── standup-brief-2026-04-19.md
├── standup-brief-2026-04-20.md
├── tasks.md
└── meeting-log.md
```

**Почему это работает:** Скил ведёт естественный журнал вашей работы без лишних усилий с вашей стороны.

## Советы

### Совет 1: Используйте стандартные маркеры задач для лучших результатов

Скил распознаёт стандартные маркеры: `- [ ]` (открыта), `- [x]` (выполнена), `TODO`, `BLOCKER`, `#task`. Чем последовательнее вы используете эти маркеры в заметках, тем точнее скил распределит элементы по секциям брифа. Нестандартные маркеры (например, `[pending]` или `WIP`) скил может не распознать — в этом случае он извлечёт пункты списков из заметок.

**Совет профи:** Добавляйте дедлайны прямо в строку задачи — например, `- [ ] Сдать отчёт (до 2026-04-21)`. Скил отсортирует задачи по дедлайну: самые срочные — наверху.

### Совет 2: Одна папка, ежедневные обновления

Скил лучше всего работает, когда заметки сосредоточены в одной папке и регулярно обновляются. Файлы, изменённые за последние 48 часов, получают приоритет — если вы обновляете `tasks.md` каждый день, результат всегда будет свежим. Если файлы разбросаны по нескольким папкам, создайте папку `standup/` и скопируйте туда ключевые файлы.

### Совет 3: Не перезаписывайте — ведите лог

Каждый бриф сохраняется с датой в имени файла (`standup-brief-2026-04-20.md`), история сохраняется автоматически. Если понадобится вспомнить, над чем вы работали в прошлый вторник — просто откройте файл. Не редактируйте сохранённый бриф — он служит логом. Если нужно что-то добавить постфактум, допишите заметку в конце файла, не меняя сгенерированное содержимое.
