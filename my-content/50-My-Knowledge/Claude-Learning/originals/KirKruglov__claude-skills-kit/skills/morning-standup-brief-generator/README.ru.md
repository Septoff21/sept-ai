# Morning Standup Brief Generator

Превращает локальные заметки и файлы задач в готовый бриф для стендапа за несколько секунд — без интеграций и подключений.

---

## Обзор

Morning Standup Brief Generator компилирует локальные `.md` и `.txt` файлы в структурированный дейли-бриф в классическом стендап-формате: **Вчера / Сегодня / Блокеры / Вопросы**. Скил сканирует рабочую папку, извлекает задачи и заметки, приоритизирует по дедлайнам и выдаёт чистый бриф. Используйте этот скил, когда нужно подготовиться к утреннему стендапу за две минуты, когда задачи хранятся в локальных файлах, а не в таск-трекере, или когда хотите вести файловый лог стендапов без установки дополнительных инструментов.

---

## Требования

- Папка с заметками и/или файлами задач (форматы `.md` или `.txt`)
  - Файлы должны содержать маркеры задач: `- [ ]` (открыта), `- [x]` (выполнена), `TODO`, `BLOCKER` и т. д.
  - Обычные заметки без маркеров тоже поддерживаются — ключевые пункты извлекаются из списков
- Внешние инструменты, коннекторы и API-ключи не нужны

**Рекомендация:** Храните дневные заметки и файлы задач в одной рабочей папке. Файлы, изменённые за последние 48 часов, получают приоритет — более старые включаются только если свежих нет.

---

## Как использовать

1. **Найдите рабочую папку**
   - Определите папку, где хранятся заметки, списки задач или проектные файлы
   - Убедитесь, что в ней есть хотя бы один `.md` или `.txt` файл

2. **Запустите скил**
   - Напишите: `"Утренний бриф для стендапа"` или `"Подготовь меня к стендапу"`
   - По-английски: `"Morning standup brief"` или `"Daily brief"`

3. **Укажите путь к папке**
   - Сообщите скилу путь к рабочей папке (например, `/Users/yourname/work-notes/`)
   - В Cowork-сессии с выбранной папкой скил использует её автоматически

4. **Получите бриф**
   - Скил сканирует файлы, извлекает выполненное, открытые задачи, блокеры и вопросы
   - Готовый бриф отображается в чате
   - Бриф также сохраняется в файл `standup-brief-YYYY-MM-DD.md` в рабочей папке

## Примеры

### Пример 1: Подготовка к стендапу из файла задач

**Входные данные:**
```
tasks.md (изменён сегодня):
- [x] Ревью PR #14 — одобрено
- [x] Отправил недельный отчёт руководителю
- [ ] Обновить документацию API (до 2026-04-21)
- [ ] Запланировать встречу по планированию Q2
- BLOCKER: Жду макеты от @Анны, без них не могу начать фронтенд
```

**Действие:** Скил читает `tasks.md`, извлекает выполненное в «Вчера», открытые задачи в «Сегодня» (по дедлайну), блокер в соответствующую секцию.

**Результат:**
```markdown
# Standup Brief — 2026-04-20

## Вчера (Выполнено)
- Ревью PR #14 — одобрено
- Отправил недельный отчёт руководителю

## Сегодня (План)
- Обновить документацию API (до 2026-04-21)
- Запланировать встречу по планированию Q2

## Блокеры
- Жду макеты от @Анны, без них не могу начать фронтенд

## Вопросы / Жду ответа
— нет

---
*Сформировано: 2026-04-20 | Источники: tasks.md*
```

### Пример 2: Несколько файлов с заметками и задачами

**Входные данные:**
```
project-notes.md (изменён сегодня):
## Итоги встречи
- Договорились перенести релиз на 25 апреля
- TODO: Подготовить чеклист релиза
- ASK @Михаил про пайплайн деплоя

work-log.md (изменён вчера):
- [x] Исправил баг в модуле авторизации
- [ ] Написать юнит-тесты для фикса
- BLOCKED: Не могу смёрджить, пока CI красный
```

**Действие:** Скил читает оба файла, дедуплицирует, строит бриф из элементов обоих файлов.

**Результат:**
```markdown
# Standup Brief — 2026-04-20

## Вчера (Выполнено)
- Исправил баг в модуле авторизации

## Сегодня (План)
- Подготовить чеклист релиза
- Написать юнит-тесты для фикса

## Блокеры
- Не могу смёрджить, пока CI красный

## Вопросы / Жду ответа
- Уточнить у @Михаила про пайплайн деплоя

---
*Сформировано: 2026-04-20 | Источники: project-notes.md, work-log.md*
```
