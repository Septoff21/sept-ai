# Руководство пользователя: Session Handoff Composer

Как переносить рабочий контекст в новую сессию без потерь.

---

## Быстрый старт

Самый быстрый способ создать хэндофф:

1. Когда сессия разрослась, напиши: "Составь хэндофф"
2. Ответь на один уточняющий вопрос, если он будет
3. Скопируй полученный markdown-блок
4. Открой новую сессию и вставь блок первым сообщением

**Результат:** Структурированное резюме сессии, которое сразу вводит Claude в курс дела.

**Время:** ~2 минуты

---

## Сценарии

### Сценарий 1: Сессия заполняется в середине задачи

**Ситуация:**
Ты продакт-менеджер, пишешь фича-спек в Cowork. Сессия началась с исследования, перешла к структуре, и сейчас черновик наполовину готов. Сессия разрослась, контекст Claude заполняется, ответы стали менее точными. Нужно продолжить черновик в свежей сессии, не потеряв ни одного решения.

**Что делать:**

1. Заметь замедление и напиши: "Составь хэндофф"
   - Ничего готовить не нужно — скил читает разговор автоматически

2. Скил может задать один вопрос
   - Например: "Критерии приёмки упоминались, но финального решения не нашёл — вы их утвердили?"
   - Ответь кратко в одном сообщении

3. Скопируй полученный блок
   - Блок содержит: что готово, что в процессе, открытые вопросы, файлы и следующие шаги
   - Ниже блока — готовая строка-открыватель для новой сессии

4. Открой новую сессию в Cowork
   - Вставь строку-открыватель: *"Продолжаем: фича-спек. Контекст ниже."*
   - Сразу за ней вставь блок хэндоффа
   - Claude прочитает его и продолжит с того места, где остановились

**Ожидаемый результат:**

Получишь блок вроде:
```
### Сделано
- Постановка проблемы написана и согласована
- User stories написаны (3 из 5)

### В процессе
- User story #4: краевой случай "нет данных" не решён

### Следующие шаги
1. Завершить user story #4 (обработать сценарий "нет данных")
2. Написать критерии приёмки
3. Добавить раздел с метриками
```

Новая сессия стартует с полным контекстом и без повторений.

**Почему это работает:** Вместо того чтобы заново вставлять старые сообщения или объяснять задачу с нуля, ты получаешь компактный структурированный брифинг, который Claude усваивает за одно сообщение.

---

### Сценарий 2: Пауза в исследовании, продолжение на следующий день

**Ситуация:**
Ты тимлид, делаешь конкурентный анализ инструментов для управления проектами. Проработал 90 минут, детально изучил 3 инструмента, нужно уйти на встречу. Хочешь продолжить завтра в новой сессии — ровно с того места, где остановился.

**Что делать:**

1. Перед закрытием сессии напиши: "Хэндофф сессии — продолжу завтра"

2. Просмотри полученный блок
   - Проверь, что все завершённые исследования в "Сделано"
   - Проверь, что оставшиеся инструменты в "Следующих шагах"
   - Если чего-то не хватает, добавь: "Также добавь: мы решили исключить Monday.com из объёма"

3. Сохрани блок в заметки или буфер обмена

4. На следующий день открой новую сессию, вставь строку-открыватель и блок, и продолжай

**Ожидаемый результат:**

Хэндофф показывает, какие инструменты готовы, что следующее, и открытый вопрос по объёму. Claude сразу продолжает исследование без вводных.

**Почему это работает:** Даже если ты закроешь приложение или вернёшься через несколько дней, хэндофф — самодостаточный брифинг. История сессии не нужна.

---

### Сценарий 3: Переключение фокуса внутри длинной сессии

**Ситуация:**
Ты контент-менеджер. Начал сессию с работы над черновиком статьи, потом переключился на редактуру другого текста. Сессия теперь содержит два рабочих потока, контекст перемешался. Хочешь вынести работу над статьёй в отдельную сессию.

**Что делать:**

1. Напиши: "Составь хэндофф только по работе над статьёй"
   - Укажи, какую тему хочешь выделить

2. Скил разделит два рабочих потока
   - Несколько тем получат отдельные разделы в хэндоффе

3. Скопируй нужный раздел
   - Возьми только часть по статье для новой сессии

**Ожидаемый результат:**

Хэндофф с чёткими разделами по двум темам. Новая сессия стартует чисто — только с нужным контекстом.

**Почему это работает:** Длинные сессии часто уходят в сторону. Хэндофф делает разделение явным, и следующая сессия начинается без лишнего шума.

## Советы

### Совет 1: Запускай раньше, не в последний момент

Не жди, пока Claude начнёт очевидно "плавать". Запускай хэндофф, когда заметишь, что ответы стали менее точными, или когда сессия идёт уже 30+ минут на сложной задаче. Ранний хэндофф — чище позднего.

**Профессиональный лайфхак:** Можно запустить хэндофф превентивно как "чекпойнт" даже если планируешь продолжить текущую сессию. Просто сохрани блок и используй по необходимости.

### Совет 2: Добавляй пропущенный контекст до копирования

После появления блока быстро прочитай его. Если чего-то критического нет (решение, принятое в начале, ограничение, которое упоминалось вскользь), добавь прямо в том же сообщении: "Также добавь в открытые вопросы: дату публикации не обсудили." Скил обновит блок.

### Совет 3: Всегда вставляй строку-открыватель вместе с блоком

Всегда вставляй строку-открыватель первой, а блок — сразу за ней. Строка-открыватель сигнализирует Claude, что это сессия-продолжение. Без неё Claude воспримет блок как сырые данные, а не как брифинг.
