# Session Handoff Composer

Переноси рабочий контекст в новую сессию без потери информации.

---

## Обзор

Session Handoff Composer извлекает ключевые элементы текущей сессии — принятые решения, задачи в работе, открытые вопросы и следующие шаги — и оформляет их в структурированный блок, готовый к вставке. Скил решает проблему потери контекста, когда длинная сессия заполняется и нужно начинать заново. Используй этот скил, когда сессия Cowork подходит к пределу контекста, когда хочешь сделать паузу и продолжить позже, или когда нужно переключить фокус в середине работы.

---

## Требования

- Активная сессия Cowork с содержательным материалом (решения, задачи или текущая работа)
- Файлы и внешние инструменты не нужны
- Скил работает полностью из контекста текущего разговора

**Рекомендация:** Используй скил, когда в сессии не менее 5–10 сообщений. Очень короткие сессии дают минимальный хэндофф.

---

## Как использовать

1. **Замечай, что сессия заполняется (или реши сделать паузу)**
   - Ничего готовить не нужно — скил читает разговор автоматически

2. **Запусти скил фразой:**
   - "Составь хэндофф" или "Подготовь handoff-документ"
   - Или на английском: "Compose handoff" или "Session handoff"

3. **Ответь на уточняющий вопрос (если будет задан)**
   - Скил может задать один вопрос, если что-то неоднозначно
   - Можно также добавить контекст, который он мог пропустить

4. **Скопируй блок хэндоффа**
   - Получишь markdown-блок, готовый к вставке как первое сообщение в новой сессии
   - К нему прилагается строка-открыватель: просто скопируй её и вставь ниже блока

Подробные сценарии использования — в [docs/USER-GUIDE.ru.md](docs/USER-GUIDE.ru.md).

## Примеры

### Пример 1: Пауза в середине задачи по написанию

**Триггер:** "Составь хэндофф"

**Результат:**
```markdown
## Session Handoff

**Тема:** Черновик письма-апдейта за Q2
**Дата:** 2026-04-19
**Статус:** в работе

### Сделано
- Согласован тон: кратко, для руководства
- Написан вступительный абзац (согласован)

### В процессе
- Тело письма: список фич ещё не финализирован

### Открытые вопросы
- Включать ли изменения по ценам в этот апдейт?

### Файлы
- q2-update-draft.md — вступление готово, тело незакончено

### Следующие шаги
1. Финализировать список фич (3–4 пункта)
2. Добавить блок по ценам, если решение — да
3. Написать заключительный призыв к действию

### Контекст для новой сессии
Пишем квартальный апдейт для руководства. Тон согласован (кратко, без жаргона). Вступление готово; тело заморожено из-за нерешённого вопроса по ценам.
```

### Пример 2: Переполнение контекста при исследовательской сессии

**Триггер:** "Хэндофф сессии — контекст почти закончился"

**Результат:**
```markdown
## Session Handoff

**Тема:** Конкурентный анализ — инструменты для управления проектами
**Дата:** 2026-04-19
**Статус:** на паузе

### Сделано
- Исследованы Asana и Notion (резюме написаны)
- Выявлено по 3 ключевых отличия для каждого

### В процессе
- Linear: исследование начато, резюме не написано

### Открытые вопросы
- Включать ли Monday.com? (не было в исходном объёме)

### Файлы
- competitive-research.md — разделы по Asana и Notion готовы

### Следующие шаги
1. Написать резюме по Linear
2. Принять решение по Monday.com
3. Составить сравнительную таблицу по всем инструментам

### Контекст для новой сессии
Конкурентный анализ для выбора PM-инструмента. Asana и Notion готовы. Linear — следующий. Таблица сравнения — после всех инструментов.
```

## Триггеры

Используй любую из этих фраз для запуска скила:

| Русский | English |
|---------|---------|
| Составь хэндофф | Compose handoff |
| Хэндофф сессии | Session handoff |
| Контекст заполняется, помоги перенести | Context is getting full, help me prepare |
| Нужно продолжить в новой сессии | I need to continue this in a new session |
| Подготовь handoff-документ | Prepare handoff document |
