# Weekly Competitor Tracker

Отслеживайте изменения у конкурентов еженедельно — без таблиц, подписок и кода.

---

## Обзор

Weekly Competitor Tracker генерирует дельта-отчёт из ваших markdown-заметок о конкурентах, показывая, что именно изменилось за неделю. Вы ведёте простую папку с `.md`-файлами (по одному на конкурента), а скил сравнивает текущие заметки с прошлонедельным снапшотом — и выделяет новые цены, изменения продукта и сдвиги в позиционировании. Используйте скил при подготовке к понедельничному стратегическому ревью, планировании роадмэпа или брифинге команды по конкурентной среде.

---

## Требования

- Папка `competitors/` с одним `.md`-файлом на конкурента
  - Файл может иметь любую структуру; заголовки H2 (`## Цены`, `## Продукт` и т.д.) улучшают качество сравнения
  - Поддерживаемый формат: обычный markdown (`.md`)
- Опционально: подпапка `competitors/snapshot/` с версиями файлов за прошлую неделю
  - Если снапшот отсутствует, скил работает в базовом режиме (сводка первого запуска, без дифа)
- Внешние инструменты, API и установки не нужны — работает полностью внутри Cowork/Claude

**Рекомендуемая структура файла конкурента:**
```markdown
## Продукт
[ключевые факты о функциях продукта]

## Цены
[тарифы, планы, изменения цен]

## Позиционирование
[слоган, позиционирование, текст на главной странице]

## Новости
[последние анонсы, статьи в блоге, новостные публикации]
```

---

## Как использовать

1. **Подготовьте папку с заметками о конкурентах**
   - Создайте папку `competitors/` с одним `.md`-файлом на конкурента (например, `acme.md`, `rival.md`)
   - Обновите заметки текущей недели в каждом файле перед запуском скила

2. **Активируйте скил, написав:**
   - «Отследи изменения конкурентов» или «Дельта-отчёт по конкурентам»
   - Укажите путь к папке `competitors/`

3. **Скил читает и сравнивает файлы**
   - Если `competitors/snapshot/` существует: выполняет побекционное сравнение с прошлой неделей
   - Если снапшот отсутствует: формирует базовую сводку (первый запуск)

4. **Изучите дельта-отчёт**
   - Файл `competitor-delta-YYYY-MM-DD.md` сохраняется в папку `competitors/`
   - Значимые изменения (цены, продукт) помечаются 🔴; незначительные — 🟡; без изменений — 🟢

5. **Обновите снапшот**
   - После ревью скопируйте текущие файлы в `competitors/snapshot/` — это установит новый базовый снапшот на следующую неделю

## Примеры

### Пример 1: Еженедельное понедельничное ревью (со снапшотом)

**Структура:**
```
competitors/
├── acme.md          ← обновлено на этой неделе
├── rival.md         ← обновлено на этой неделе
└── snapshot/
    ├── acme.md      ← версия прошлой недели
    └── rival.md     ← версия прошлой недели
```

**Действие:** Напишите «Дельта-отчёт по конкурентам» и укажите путь к папке `competitors/`.

**Фрагмент вывода:**
```markdown
# Дельта-отчёт по конкурентам — 2026-04-21

**Период:** 2026-04-14 → 2026-04-21
**Конкурентов отслеживается:** 2
**Изменений обнаружено:** 2 (Высокая: 1, Средняя: 1, Низкая: 0)

## Acme

**Итог:** 🔴 Значимые изменения

### Что изменилось
| Раздел | Изменение | Значимость |
|--------|-----------|------------|
| Цены | Добавлен тариф Starter за $49/мес | Высокая |

### Без изменений
- Продукт: изменений не обнаружено
```

---

### Пример 2: Первый запуск (без снапшота — базовый режим)

**Структура:**
```
competitors/
├── acme.md
└── rival.md
```
*(папки snapshot/ нет)*

**Действие:** Напишите «Отследи изменения конкурентов».

**Фрагмент вывода:**
```markdown
# Дельта-отчёт по конкурентам — 2026-04-21

⚠️ Первый запуск — снапшот не найден
Это ваша базовая сводка. Чтобы включить трекинг изменений на следующей неделе,
скопируйте все файлы из `competitors/` в `competitors/snapshot/`.

## Acme

### Базовая сводка
**Продукт:** Инструмент совместной работы команды с редактированием в реальном времени.
**Цены:** Бесплатный тариф + Pro за $12/пользователь/мес + Enterprise по запросу.
**Позиционирование:** «Работайте лучше вместе.»
```
