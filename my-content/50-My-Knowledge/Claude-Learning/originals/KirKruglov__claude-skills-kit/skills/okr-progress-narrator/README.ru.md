# OKR Progress Narrator

**Преобразуй сырые OKR-данные в читаемый апдейт для стейкхолдеров — автоматически.**

[![Skills](https://img.shields.io/badge/claude--skills--kit-okr--progress--narrator-blue)](https://github.com/KirKruglov/claude-skills-kit)

---

## Что делает

OKR Progress Narrator принимает сырые OKR-данные — таблицу, список, CSV или текст в чате — и преобразует их в чистый нарративный апдейт, готовый к отправке стейкхолдерам.

Вместо ручного написания отчёта по метрикам ты получаешь:
- **Executive summary** в 2–4 предложениях
- **Нарративный абзац по каждой цели** с объяснением прогресса и рисков
- **Таблицу KR** с % прогресса и цветовым статусом
- **Блок ключевых рисков** по at-risk и off-track KR
- **Следующие шаги** (если в данных есть запланированные действия)

Результат: `okr-update-YYYY-MM-DD.md`

## Форматы входных данных

Скил принимает OKR-данные в любом из форматов:

| Формат | Пример |
|--------|--------|
| Markdown-таблица | `\| Цель \| KR \| Текущее \| Цель \|` |
| Список с отступами | `О1: Рост выручки` / `- КР1.1: ARR 10M` |
| CSV / TSV файл | Любой файл с колонками Цель, KR, Текущее, Цель |
| Свободный текст | Описание целей и прогресса в произвольной форме |

**Опциональные параметры:**
- Период: `Q2 2026`, `Апрель`, `Спринт 12`
- Аудитория: `CEO`, `совет директоров`, `команда` (влияет на глубину нарратива)
- Фокус: `только at-risk` или `только highlights`

## Пример

**Входные данные (вставлены в чат):**
```
OKR Q1 2026

Цель 1: Вырасти MRR до 200K $
- KR1: Новый MRR — текущее 42K, цель 50K, заметки: пайплайн сильный
- KR2: Churn — текущее 3.2%, цель <2.5%, заметки: удержание под угрозой

Цель 2: Запустить мобильное приложение
- KR1: Релиз в App Store — выполнено ✅
- KR2: 1000 DAU на 4-й неделе — текущее 680, цель 1000
```

**Фрагмент результата:**
```markdown
# OKR Progress Update — Q1 2026
**Подготовлено:** 2026-04-18

## Executive Summary
1 из 2 целей в зелёной зоне. Мобильное приложение запущено и набирает 
аудиторию — DAU на пути к таргету. Рост MRR идёт в темпе, однако churn 
превышает пороговое значение и требует приоритетного внимания.

## По целям

### Вырасти MRR до 200K $ — 🟡 Под угрозой
Новый MRR составляет 84% от квартального подтаргета при сильном пайплайне. 
Вместе с тем churn превысил целевой порог, что может нивелировать 
приобретение клиентов, если не будет устранён в этом квартале.
...
```
