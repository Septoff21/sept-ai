# Weekly AI Workflow Review User Guide

Learn how to turn your weekly Claude interactions into actionable patterns and reusable prompt templates.

---

## Quick Start

Here's the fastest way to get your weekly review:

1. Jot down 3–10 tasks you delegated to Claude this week (one line each is enough)
2. Say: "Weekly AI workflow review" and paste your notes
3. Get back a structured report with patterns, wins, and prompt templates for next week

**Result:** A clear reflection on your AI workflow — what to repeat, what to fix.

**Time:** ~3 minutes

---

## Scenarios

### Scenario 1: End-of-Week Reflection with Rough Notes

**Situation:**
You are a product manager who uses Claude daily but has never tracked your interactions formally. On Friday afternoon, you want to spend 10 minutes reviewing your week of AI usage to understand what worked and build better habits. You only have rough mental notes, not a structured log.

**What to do:**

1. Take 2–3 minutes to write down the tasks you delegated to Claude this week
   - One line per task: what you asked for and roughly what happened
   - Example: "Asked Claude to write release notes — worked fine, used them"
   - Example: "Tried to get Claude to draft an investor update — spent 30 min editing"
   - Don't worry about formatting; any list works

2. Trigger the skill by saying: "Weekly AI workflow review"
   - Paste your rough notes when prompted
   - The skill will parse the list and identify task types, patterns, and issues

3. Read the patterns section first
   - See which task types dominated your week
   - Note the successful / rework ratio

4. Focus on the improvement recommendations
   - For each pain point, the skill provides a specific prompt fix
   - Copy the fix and use it next time you face the same task

5. Save the prompt templates section
   - Copy the 1–2 recommended templates into a personal notes file or sticky note
   - Use them as a starting point next week

**Expected result:**

You receive a reflection report showing:
- Your dominant task types (e.g., "Content writing: 60% of week")
- 1–2 successful interactions with their prompt patterns
- 1–2 pain points with specific improvement recommendations
- 1–2 reusable prompt templates

You spend 5 minutes reading the report and walk away with 2 concrete improvements to try next week.

**Why this works:** Even rough notes give the skill enough signal to identify patterns. You don't need a formal log — 2–3 minutes of memory recall is sufficient to get actionable insights.

### Scenario 2: Tracking Prompt Quality Over Multiple Weeks

**Situation:**
You are a team lead who uses Claude for recurring tasks: weekly status reports, meeting summaries, and team update emails. You've been using the same prompts for a month and suspect they could be better. You want to compare what's working against what keeps needing edits — and build a proper prompt library for your recurring tasks.

**What to do:**

1. Keep a simple log during the week
   - Create a note file or doc: one entry per Claude interaction
   - Format: task | prompt (or brief summary) | outcome (used / edited / discarded)
   - Takes 30 seconds per entry; log immediately after each interaction

2. At week's end, trigger the skill: "Review my Claude interactions this week"
   - Upload or paste your log
   - The skill identifies recurring tasks and flags which prompts need improvement

3. Focus on the "Recommended Prompt Templates" section
   - The skill generates templates for your top recurring task types
   - Compare the template to what you currently use — adjust where you see gaps

4. Update your prompt library
   - Save the new templates to a personal reference file
   - Replace old, underperforming prompts with the new templates
   - Use the updated prompts next week and note results

5. Repeat weekly for 3–4 weeks
   - Track whether rework rate decreases
   - Retire templates that still underperform after 2 tries

**Expected result:**

After 3–4 weeks of weekly reviews, you have:
- A library of 3–5 proven prompt templates for your recurring tasks
- A clear understanding of which task types Claude handles well vs. needs guidance
- Measurably less time spent editing Claude's output

**Why this works:** Weekly cadence turns individual interactions into a feedback loop. The skill acts as a coach — it identifies patterns you miss when you're heads-down in the work.

### Scenario 3: Diagnosing a Bad Week with Claude

**Situation:**
You are a content creator. This week, almost nothing Claude produced worked as expected — you edited or discarded most outputs. You're frustrated and wondering if you're prompting incorrectly, or if Claude just can't handle your use cases. You want a diagnosis before giving up on AI-assisted workflows.

**What to do:**

1. Write down all the tasks where Claude failed you this week
   - Include: what you asked for, what Claude produced, what you changed
   - Be specific: "Asked for a casual LinkedIn post about product launch; got formal corporate text; rewrote entirely"

2. Trigger the skill: "Help me reflect on how I used Claude this week"
   - Paste your frustration log (it doesn't need to be polished)
   - The skill will identify the root causes of the failures

3. Read the "Areas for Improvement" section carefully
   - Each pain point comes with a diagnosis: vague request / missing context / scope too broad / wrong tone
   - Read the specific recommendation for each

4. Test one improved prompt immediately
   - Pick the most common pain point from the report
   - Use the recommended template on a new similar task right now
   - Compare the output to what you got before

5. If the new prompt works: update your approach for that task type
   - If it still doesn't work: note it and try once more with added constraints

**Expected result:**

You receive a diagnosis report showing that 80% of your pain points trace to one root cause (e.g., "missing tone / audience specification in all content requests"). You get 1–2 corrected prompt templates.

You test the improved prompt and get a usable output on the first try — confirming the issue was the prompt, not Claude's capability.

**Why this works:** The skill turns a frustrating week into structured learning. Instead of "Claude doesn't work for me," you get "here's exactly what to change."

## Tips

### Tip 1: Log Outcomes in Real Time — Don't Wait Until Friday

The most common mistake is trying to recall everything at week's end. Instead, keep a running note (phone, sticky note, Notion page) and log each interaction immediately: task, prompt summary, outcome. This takes 20–30 seconds and dramatically improves the quality of the weekly review.

**Pro tip:** Use a simple 3-column format: `Task | Prompt | Outcome`. Even 3 words per column is enough for the skill to work with.

### Tip 2: Include the Actual Prompt (or a 1-Sentence Summary)

The skill generates better prompt templates when it can see what you actually wrote. If you log "asked Claude to write an email," the template will be generic. If you log "write a follow-up email for enterprise sales, no context given," the template will be specific and fixable.

**Pro tip:** Copy the first sentence of your prompt into the log. That's enough to see the pattern.

### Tip 3: Use the "Takeaway" Section to Set One Habit for Next Week

The Takeaway is a 1–2 sentence summary at the end of the report. Use it to set exactly one behavior change for next week — not a list of five improvements. One concrete habit (e.g., "always specify audience and format in content requests") applied consistently beats five rules you'll forget by Monday.

**Pro tip:** Paste the Takeaway into a calendar reminder for Monday morning. Read it before your first Claude interaction of the week.
