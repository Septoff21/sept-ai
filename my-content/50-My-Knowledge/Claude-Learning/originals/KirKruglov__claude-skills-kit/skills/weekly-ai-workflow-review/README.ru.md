# Weekly AI Workflow Review

Превратите неделю взаимодействий с Claude в конкретные инсайты: паттерны, победы и шаблоны промптов на следующую неделю.

---

## Обзор

Weekly AI Workflow Review анализирует ваши заметки о задачах, делегированных Claude, и формирует структурированный отчёт о рефлексии. Скил выявляет повторяющиеся типы задач, успешные промпты и взаимодействия, потребовавшие доработки — и генерирует многоразовые шаблоны промптов на основе того, что сработало. Используйте скил, когда хотите понять, как вы применяете AI в течение недели, улучшить навыки промптинга со временем или построить личную библиотеку эффективных промптов без технических инструментов.

---

## Требования

- Еженедельный лог задач в формате markdown или обычного текста
  - Подходит структурированный (нумерованный список, markdown-буллеты) и неструктурированный формат (абзацы, заметки)
  - Минимум: описания задач и что вы просили Claude сделать
  - Рекомендуется: пометить результат каждой задачи (использован без правок / отредактирован / отклонён)
- Внешние инструменты, API и плагины не нужны — работает с обычным текстом в чате

**Рекомендация:** Фиксируйте минимум 5 задач в неделю для выявления значимых паттернов. Даже грубые заметки работают: «попросил Claude написать письмо для продукта, пришлось переписать тон».

---

## Как использовать

1. **Соберите еженедельный лог задач Claude**
   - Ведите простой список в течение недели: задача, промпт (или его пересказ), результат
   - Или восстановите по памяти в конце недели — даже грубые описания подойдут

2. **Запустите скил**
   - Скажите: «Еженедельный обзор AI-задач» или «Проанализируй мои задачи Claude за неделю»
   - По-английски: «Weekly AI workflow review» или «Review my Claude interactions this week»

3. **Предоставьте лог задач**
   - Вставьте текст прямо в чат или загрузите markdown-файл
   - Специальный формат не нужен — скил разбирает структурированный и неструктурированный ввод

4. **Изучите отчёт**
   - Вы получите структурированную рефлексию: паттерны, победы, зоны улучшения и шаблоны промптов
   - По желанию — попросите сохранить отчёт как `weekly-ai-review-YYYY-MM-DD.md`

## Примеры

### Пример 1: Простой недельный лог (без отметок о результатах)

**Ввод:**
```
Неделя 14 апреля:
- Попросил Claude написать письмо-обновление о продукте для команды
- Claude резюмировал 3 поста конкурентов
- Использовал Claude для 5 идей постов в LinkedIn
- Claude помог с отчётом по статусу проекта
- Перерабатывал описание вакансии с Claude
```

**Действие:** Скил классифицирует задачи по типам, определяет «Контент/тексты» как доминирующий тип, отмечает отсутствие данных о результатах и генерирует шаблоны промптов для двух наиболее частых типов задач.

**Вывод:**
```markdown
## Еженедельный обзор AI-рабочего процесса
**Период:** Неделя 14 апреля  **Задач проанализировано:** 5

### Паттерны недели
- Доминирующий тип: Контент / Тексты (4 задачи, 80%)
- Повторяющиеся темы: письма, посты LinkedIn
- Данные о результатах: отсутствуют — добавьте отметки на следующей неделе

### Рекомендованные шаблоны промптов
1. Для написания писем: «Напиши [тип] письмо для [аудитории]. Тон: [формальный/casual]. Длина: [X предложений]. Ключевые пункты: [список]. Не включай: [ограничения].»
2. Для резюмирования: «Резюмируй следующий [тип контента] в [N] буллетах. Фокус: [угол]. Аудитория: [кто прочитает].»

### Итог
Написание контента доминирует в вашей работе с Claude. Добавьте отметки о результатах на следующей неделе для полноценного анализа паттернов.
```

### Пример 2: Детальный лог с результатами

**Ввод:**
```
- Черновик changelog → использован без правок. Промпт: «Напиши changelog для фич v2.3: [список]. Аудитория: разработчики. Формат: markdown-буллеты.»
- Executive summary за Q1 → переписал 70%. Промпт: «Резюмируй результаты Q1.» Слишком расплывчато.
- 3 вопроса для интервью UX → использовал 2 из 3. Приемлемо.
- Cold outreach письмо → отклонил. Промпт: «Напиши холодное письмо.» Нет контекста.
- Дайджест из 4 Slack-тредов → использован без правок. Промпт: «Резюмируй эти Slack-сообщения в дайджест для команды: [вставка].»
```

**Действие:** Скил выявляет 2 успешных промпта (changelog, дайджест), 2 проблемных взаимодействия и генерирует конкретные рекомендации по улучшению.

**Вывод:**
```markdown
## Еженедельный обзор AI-рабочего процесса
**Период:** Текущая неделя  **Задач проанализировано:** 5

### Паттерны недели
- Доминирующий тип: Контент / Тексты (5 задач)
- Успешных взаимодействий: 3 (60%)
- Задач с доработкой: 2 (40%)

### Что работало хорошо
- **Написание контента:** Changelog использован без правок. Паттерн: явный формат + аудитория + список контента.
- **Резюмирование:** Дайджест принят без правок. Паттерн: чёткий формат вывода + полный контекст вставлен.

### Зоны улучшения
- **Executive summary:** Переписан на 70%. Причина: расплывчатый запрос — нет формата, длины, аудитории. Рекомендация: «Резюмируй результаты Q1 в [N] абзацах. Аудитория: [руководство]. Фокус на: [выручка/рост/риски]. Формат: проза, без буллетов.»
- **Cold email:** Отклонён. Причина: нет контекста — роль получателя, компания, цель не указаны. Рекомендация: «Напиши холодное письмо для [должность] из [тип компании]. Цель: [результат]. Тон: [стиль]. Длина: до 150 слов. Контекст: [краткая предыстория].»

### Рекомендованные шаблоны промптов
1. Для отчётов/резюме: «Резюмируй [контент] в [N абзацах/буллетах]. Аудитория: [кто]. Фокус: [ключевой угол]. Формат: [проза/буллеты/таблица].»
2. Для писем: «Напиши [тип] письмо для [роль получателя]. Цель: [результат]. Тон: [стиль]. Длина: [X слов]. Контекст: [предыстория].»

### Итог
Промпты с явным форматом + аудиторией + ограничениями стабильно дают результат без правок. На следующей неделе: применить шаблон к каждому запросу на резюме.
```
